系统建模

System Modeling — 刻画程序功能语义、输入域与关键判定逻辑，为风险分析提供结构基础。

第一部分 功能语义 输入域 判定逻辑

目的与定位

系统建模是风险驱动测试的第一步，其核心目的是为后续的风险分析提供结构基础。

关键理解：建模用于定位可能引发风险的结构位置，而不是穷尽所有行为细节。

在软件系统中，风险通常由三个维度共同形成：

  • 功能语义 — 功能行为与预期语义不一致，决定风险后果严重程度
  • 输入域 — 参数组合或边界条件决定风险触发入口
  • 关键判定逻辑 — 条件分支与状态路径决定风险传播路径

语义决定风险价值，结构决定风险分布与触达方式。

功能语义识别

简述被测程序的核心行为，包括：

  • 默认输出行为 — 程序在无特殊选项时的默认行为
  • 选项对输出行为的影响 — 各选项如何改变程序行为
  • 参数与输出关系 — 输入参数如何映射到输出
  • 换行与转义处理规则 — 特殊字符的处理方式
要求：描述不超过 10 行文字；描述行为含义，而非实现细节。

示例：echo 命令的功能语义

echo 用于将输入参数输出到标准输出。默认情况下，所有参数按顺序输出，并在末尾添加换行。各选项影响如下：

选项行为
-n取消末尾换行
-e启用反斜杠转义字符解析
-E显式关闭转义解析（默认行为）

启用转义解析时的关键转义字符：

  • \n — 换行（结构变化行为）
  • \t — 制表符（字符替换行为）
  • \c — 终止输出，忽略后续字符，不再输出换行（控制行为）

多个参数之间默认以空格分隔输出。

输入域识别

描述被测程序的输入构成，可采用结构图或分层列表形式。

输入结构

command [option] [string]...

输入构成

选项集合

  • -n 取消换行
  • -e 启用反斜杠转义解析
  • -E 显式禁用转义解析（默认行为）

普通参数

  • 字符串
  • 可能包含转义字符
  • 可能为空

输入组合示例

组合示例说明
纯参数echo hello最简单场景
单选项 + 参数echo -n hello选项影响输出格式
转义选项 + 参数echo -e "a\nb"启用转义解析
多选项 + 参数echo -n -e "hello"选项组合
多参数echo "a" "b"参数分隔处理

关键判定逻辑识别

结合代码或行为说明，识别 3-5 个关键判定逻辑。每项用 1-2 行说明其对行为的影响。

编号判定逻辑对行为的影响
1 是否存在 -n 选项 决定输出末尾是否添加换行
2 是否启用 -e 转义 决定是否对字符串中的反斜杠序列进行解析
3 是否出现 \c 终止后续输出并取消换行
4 是否存在多个参数 多个参数之间是否插入空格

Echo 完整建模示例

以下展示使用 BusyBox 中的 echo 工具进行系统建模的完整过程。

echo 功能语义识别

  1. 默认将所有参数按顺序输出到标准输出，末尾添加换行
  2. -n 取消末尾换行
  3. -e 启用反斜杠转义字符解析
  4. -E 显式关闭转义解析（默认行为等价）
  5. \n 换行、\t 制表符、\c 终止输出
  6. 多参数之间以空格分隔

echo 输入域识别

输入结构：echo [option] [string]...

选项集合：-n、-e、-E

普通参数：字符串，可能包含转义字符，可能为空

典型组合：

  • echo hello — 纯参数
  • echo -n hello — 单选项
  • echo -e "a\nb" — 转义选项
  • echo -n -e "hello" — 多选项组合
  • echo "a" "b" — 多参数

echo 关键判定逻辑

  1. 是否存在 -n：决定输出末尾是否添加换行
  2. 是否启用 -e：决定是否对反斜杠序列进行解析
  3. 是否出现 \c：终止后续输出并取消换行
  4. 是否存在多个参数：决定参数间是否插入空格

建模总结

完成系统建模后，需要回答以下三个问题：

  1. 选项组合 — 多个选项同时使用时的行为一致性
  2. 转义字符类型 — 不同转义序列的解析正确性
  3. 参数数量 — 无参数、单参数、多参数的不同处理逻辑
  1. 是否启用转义 — 决定进入转义解析路径或直接输出
  2. 是否取消换行 — 决定输出末尾行为
  3. 是否触发 \c — 决定是否提前终止输出
  1. 选项解析逻辑 — 选项识别与组合处理
  2. 转义解析逻辑 — 各类转义序列的处理分支
  3. 输出控制逻辑 — 换行、终止等输出行为控制
