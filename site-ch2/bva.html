Ch2 › 边界值分析

边界值分析

Boundary Value Analysis — 等价类划分与边界值分析的2-value和3-value方法。

BVA 等价类 边界值

等价类划分

基本思想

等价类划分基于一个假设：在同一等价类中的输入值，程序的行为应该是相似的。因此，从每个等价类中选取一个代表值进行测试即可。

等价类与ISP的关系

ISP中的"块"本质上就是等价类。每个特征的每个块代表一组等价的输入值。边界值分析（BVA）在等价类的基础上，特别关注等价类的边界——因为缺陷往往集中在边界处。

经验法则： 大量缺陷发生在输入域的边界处，而非内部。例如数组索引的0和length-1、循环的起始和终止条件、数值范围的上下限等。

2-value边界值分析

定义： 对每个等价类，取边界值和相邻等价类的边界值作为测试输入。

示例

输入域 {1..16} 划分为三个等价类：

等价类范围边界值
EC1{1..6}1, 6
EC2{7..13}7, 13
EC3{14..16}14, 16

2-value BVA 测试值：

边界值

每个等价类的边界：1, 6, 7, 13, 14, 16

加上域外边界

输入域外的相邻值：0, 17

测试值集合：{0, 1, 6, 7, 13, 14, 16, 17} — 共 8 个测试值

3-value边界值分析

定义： 对每个边界值 x，取 x-1, x, x+1 三个值作为测试输入。

示例

同样的输入域 {1..16}，等价类 {1..6}, {7..13}, {14..16}：

边界值：0, 1, 6, 7, 13, 14, 16, 17

对每个边界值取三邻域：

3-value 能发现更多缺陷

考虑以下代码：

if (x >= 15) {
    // 正确：x大于等于15时执行
}
// 缺陷：误写为 x == 15

2-value BVA

测试值：14, 16

✅ 能发现此缺陷（16不满足x==15但应满足x>=15）

3-value BVA

测试值：14, 15, 16

✅ 能发现此缺陷，并且能精确识别边界位置

考虑另一个缺陷：

if (x > 15) {
    // 缺陷：误写为 x >= 15（应为严格大于）
}

2-value 测试值 14, 16 无法区分 >15 和 >=15，但 3-value 的测试值 15 能发现此缺陷。

连续域处理

问题

实数域是稠密的——任意两个实数之间都有无穷多个实数。因此没有"紧邻"的值。

解决方案

由问题性质或计算机表示确定最小增量：

领域最小增量示例
价格0.01（分）9.99, 10.00, 10.01
温度0.1（度）36.5, 36.6, 36.7
重量0.1（kg）79.9, 80.0, 80.1
浮点数机器精度取决于数据类型
实践建议： 在需求分析阶段就应该确定每个数值参数的精度和有效范围，这有助于定义边界值。

2-value vs 3-value 对比

2-value BVA

  • 测试值较少（~2n个边界）
  • 能发现大多数边界缺陷
  • 实现简单
  • 适合快速测试

3-value BVA

  • 测试值较多（~3n个边界）
  • 能发现更多边界缺陷
  • 能精确识别边界位置
  • 适合关键系统

BVA 练习

电梯最大承重280kg。等价类：{≤280}（正常）, {>280}（超载）。2-value BVA测试值：279, 280, 281。3-value：279, 280, 281（边界值本身已在集合中）。

有效范围1900-2020。2-value BVA测试值：1899, 1900, 2020, 2021。3-value：1899, 1900, 1901, 2019, 2020, 2021。

购物金额分档：0-100无折扣，101-500九折，501+八折。边界值：0, 100, 101, 500, 501。2-value：0, 100, 101, 500, 501。3-value：-1, 0, 1, 99, 100, 101, 102, 499, 500, 501, 502。

PIN为4位数字（0000-9990000-9999）。边界值主要关注长度：3位、4位、5位。等价类：{长度≠4}, {长度=4且格式正确}。
