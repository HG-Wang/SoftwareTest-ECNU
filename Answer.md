# HW5 — 基于语法的变异测试：RIPR 条件分析

## 背景知识

### RIPR 模型

变异测试中判断一个测试用例是否能"杀死"某个变体，需要满足 **RIPR** 四个条件：

| 条件 | 含义 | 说明 |
|------|------|------|
| **R — Reachability（到达）** | 被变异的语句必须被执行 | 测试输入需使程序执行路径经过变异点 |
| **I — Infection（感染）** | 变异语句执行后程序状态必须与原程序不同 | 变体在变异点处产生了不同于原程序的状态 |
| **P — Propagation（传播）** | 不同的状态必须传播到最终输出 | 感染的状态差异不能被后续代码"修复" |
| **R — Revealability（可观察性）** | 测试者必须能观察到输出差异 | 输出的不同之处需要被测试者检测到 |

**强杀死（Strongly Kill）**：满足 R + I + P + R，即变体的最终输出与原程序不同。
**弱杀死（Weakly Kill）**：仅满足 R + I，即变异点之后的程序状态不同（不要求传播到输出）。

## Task 1：Min() 函数变体分析

### 原始程序

```cpp
int Min(int A, int B) {
    int minVal;
    minVal = A;
    if (B < A) {
        minVal = B;
    }
    return (minVal);
}
```

### 程序控制流

```
[1] minVal = A
[2] if (B < A)
      ├── true  → [3] minVal = B
      └── false → (skip)
[4] return (minVal)
```

---

### 变体 Δ2：`if (B > A)` 替换 `if (B < A)`

变异点在条件判断处，将 `B < A` 改为 `B > A`。

| RIPR 条件 | 分析 | 表达式 |
|-----------|------|--------|
| **Reachability** | 条件判断语句在每次调用时都会被执行，无前置分支 | **无条件（始终到达）** |
| **Infection** | 原程序和变体的分支判定不同 → 执行路径不同。原程序在 `B < A` 时进入 if 分支，变体在 `B > A` 时进入。需要两个条件的布尔值不同。 | **A ≠ B** |
| **Propagation** | 分支走向不同会导致 minVal 的值不同。若 `B < A`：原程序 minVal = B，变体 minVal = A。若 `B > A`：原程序 minVal = A，变体 minVal = B。两种情况下输出均不同。 | **A ≠ B**（与 Infection 相同） |

**杀死变体的测试用例：**

| A | B | 原程序输出 | 变体输出 | 说明 |
|---|---|-----------|---------|------|
| **5** | **3** | 3 | 5 | B < A，原程序进入分支取 B，变体不进入取 A |
| **2** | **7** | 2 | 7 | B > A，原程序不进入分支取 A，变体进入取 B |

任选一组即可杀死变体，例如 **(A=5, B=3)**。

---

### 变体 Δ4：`Bomb()` 替换 `minVal = B`

变异点在 if 分支体内，将 `minVal = B` 替换为 `Bomb()`（抛出异常）。

| RIPR 条件 | 分析 | 表达式 |
|-----------|------|--------|
| **Reachability** | 必须进入 if 分支体，即条件 `B < A` 为真 | **B < A** |
| **Infection** | `Bomb()` 抛出异常，程序状态完全不同于正常执行的 `minVal = B` | **B < A**（进入分支即感染） |
| **Propagation** | 异常会直接传播，程序终止并抛出异常而非返回正常值。输出完全不同（异常 vs 正常返回值） | **B < A**（异常自动传播） |

**杀死变体的测试用例：**

| A | B | 原程序输出 | 变体输出 | 说明 |
|---|---|-----------|---------|------|
| **5** | **3** | 3 | 异常 (Bomb) | B < A 成立，进入分支后原程序正常返回 3，变体抛出异常 |

---

### 变体 Δ5：`minVal = A` 替换 `minVal = B`

变异点在 if 分支体内，将 `minVal = B` 替换为 `minVal = A`。

| RIPR 条件 | 分析 | 表达式 |
|-----------|------|--------|
| **Reachability** | 必须进入 if 分支体，即条件 `B < A` 为真 | **B < A** |
| **Infection** | 原程序执行 `minVal = B`，变体执行 `minVal = A`。分支进入条件已要求 `B < A`，因此 `B ≠ A`，状态不同。 | **B < A**（蕴含 A ≠ B） |
| **Propagation** | return 语句返回 minVal。原程序返回 B，变体返回 A。由于 B < A 即 B ≠ A，输出不同。 | **B < A**（蕴含输出不同） |

**杀死变体的测试用例：**

| A | B | 原程序输出 | 变体输出 | 说明 |
|---|---|-----------|---------|------|
| **5** | **3** | 3 | 5 | B < A 成立，原程序 minVal = 3，变体 minVal = 5 |
| **10** | **1** | 1 | 10 | B < A 成立，原程序 minVal = 1，变体 minVal = 10 |

任选一组即可，例如 **(A=5, B=3)**。

---

### 变体 Δ6：`minVal = failOnZero(B)` 替换 `minVal = B`

变异点在 if 分支体内，将 `minVal = B` 替换为 `minVal = failOnZero(B)`。

`failOnZero(B)` 是一个检测函数：当 `B = 0` 时抛出异常（或返回不同于 B 的值）；当 `B ≠ 0` 时返回 B 本身。

| RIPR 条件 | 分析 | 表达式 |
|-----------|------|--------|
| **Reachability** | 必须进入 if 分支体，即条件 `B < A` 为真 | **B < A** |
| **Infection** | 需要 `failOnZero(B) ≠ B`。当 B ≠ 0 时，`failOnZero(B) = B`，无感染。当 B = 0 时，`failOnZero(0)` 抛出异常或返回不同值，产生感染。 | **B = 0**（且 B < A，即 A > 0） |
| **Propagation** | 当 B = 0 时，`failOnZero(0)` 抛出异常，直接传播到输出。或返回值不同于 0，导致 minVal 不同，传播到 return 语句。 | **B = 0**（异常自动传播或输出不同） |

**杀死变体的测试用例：**

| A | B | 原程序输出 | 变体输出 | 说明 |
|---|---|-----------|---------|------|
| **1** | **0** | 0 | 异常 (failOnZero) | B < A 成立，B = 0 触发 failOnZero 异常 |
| **5** | **0** | 0 | 异常 (failOnZero) | 同上 |

---

### Task 1 总结

| 变体 | 变异点 | Reachability | Infection | Propagation | 测试用例 (A, B) |
|------|--------|-------------|-----------|-------------|-----------------|
| **Δ2** | 条件 `B < A` → `B > A` | 无条件 | A ≠ B | A ≠ B | **(5, 3)** |
| **Δ4** | `minVal = B` → `Bomb()` | B < A | B < A | B < A | **(5, 3)** |
| **Δ5** | `minVal = B` → `minVal = A` | B < A | B < A | B < A | **(5, 3)** |
| **Δ6** | `minVal = B` → `minVal = failOnZero(B)` | B < A | B = 0 | B = 0 | **(1, 0)** |

## Task 2：findVal() 方法变体分析

### 原始程序

```java
// Effects: If numbers null throw NullPointerException
//    else return LAST occurrence of val in numbers[]
//    If val not in numbers[] return -1
public static int findVal(int numbers[], int val) {
    int findVal = -1;
    for (int i = 0; i < numbers.length; i++)
        if (numbers[i] == val)
            findVal = i;
    return (findVal);
}
```

### 变体描述

将循环初始化 `i = 0` 改为 `i = 0 + 1`（即 `i = 1`），导致跳过 `numbers[0]` 的检查。

```
// 原始:   for (int i = 0;      i < numbers.length; i++)
// 变体:   for (int i = (0+1);  i < numbers.length; i++)
//                       ↑ 即 i = 1
```

### (a) 是否存在测试输入使得不到达变体？

**分析：** 变异点是 for 循环的初始化语句 `int i = 0` → `int i = 1`。在 Java 中，for 循环的初始化部分在进入循环时**无条件执行**。函数被调用后，执行流必然经过 for 循环头。

- 若 `numbers = null`：在 `numbers.length` 处抛出 NPE，但初始化 `i = 0/1` 已经执行完毕（NPE 发生在条件检查阶段）。
- 若 `numbers` 非空：初始化正常执行。

**结论：不可能。** 无论什么测试输入，for 循环初始化语句始终会被执行，变体始终被到达。

---

### (b) 是否存在测试输入使得到达但不感染？

**分析：** 到达变体后，原程序中 `i = 0`，变体中 `i = 1`。此时变量 `i` 的值已经不同 — 这就是程序状态的差异。根据 Infection 的定义（变异语句执行后程序状态不同），只要执行到该语句，状态就一定不同。

**结论：不可能。** `i = 0` 和 `i = 1` 是不同的值，到达即感染。

---

### (c) 是否存在测试输入使得感染但不传播？

**分析：** 感染已满足（i 值不同）。要使差异不传播到输出，需要变体和原程序的**返回值相同**。

变体跳过了 `numbers[0]` 的检查。如果 `numbers[0] ≠ val`，则即使原程序检查了 `numbers[0]`，也不会设置 `findVal = 0`（因为不相等）。此时跳过 `numbers[0]` 不影响最终结果。

**测试用例：** `numbers = [1, 2, 3], val = 2`

| 步骤 | 原程序 (i=0起) | 变体 (i=1起) |
|------|---------------|-------------|
| i=0 | numbers[0]=1 ≠ 2, 跳过 | (不执行) |
| i=1 | numbers[1]=2 == 2, findVal=1 | numbers[1]=2 == 2, findVal=1 |
| i=2 | numbers[2]=3 ≠ 2, 跳过 | numbers[2]=3 ≠ 2, 跳过 |
| **返回** | **1** | **1** |

**结论：可以。** `numbers = [1, 2, 3], val = 2` 满足感染但不传播（输出相同，均为 1）。

---

### (d) 是否存在测试输入能够杀死变体？

**分析：** 要杀死变体，需要变体的输出与原程序不同。变体跳过了 `numbers[0]` 的检查，因此需要 `numbers[0] == val` 且 `val` 在数组中**最后一次**出现在 index 0（即 `val` 不出现在 index 1 及之后的位置）。

**测试用例 1：** `numbers = [5], val = 5`

| 步骤 | 原程序 | 变体 |
|------|--------|------|
| i=0 | numbers[0]=5 == 5, findVal=0 | (不执行) |
| 循环结束 | 返回 **0** | 返回 **-1** |

输出不同（0 vs -1），变体被杀死。

**测试用例 2：** `numbers = [5, 3, 7], val = 5`

| 步骤 | 原程序 | 变体 |
|------|--------|------|
| i=0 | numbers[0]=5 == 5, findVal=0 | (不执行) |
| i=1 | numbers[1]=3 ≠ 5 | numbers[1]=3 ≠ 5 |
| i=2 | numbers[2]=7 ≠ 5 | numbers[2]=7 ≠ 5 |
| **返回** | **0** | **-1** |

输出不同（0 vs -1），变体被杀死。

**结论：可以。** `numbers = [5], val = 5` 杀死变体。

### Task 2 总结

| 问题 | 是否可能 | 测试用例 | 说明 |
|------|---------|---------|------|
| **(a) 不到达** | ❌ 不可能 | — | for 循环初始化始终执行 |
| **(b) 到达不感染** | ❌ 不可能 | — | i=0 vs i=1 状态始终不同 |
| **(c) 感染不传播** | ✅ 可能 | `[1,2,3], val=2` | numbers[0]≠val，跳过不影响结果 |
| **(d) 杀死变体** | ✅ 可能 | `[5], val=5` | numbers[0]=val 且是唯一出现位置 |
