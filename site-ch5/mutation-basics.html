变异测试基础

Mutation Testing — 通过系统地修改（变异）软件构件来生成测试用例、评估测试集质量的测试方法。被广泛认为是测试的"黄金标准"。

5.1.2 变异测试 变体 杀死变体

什么是变异？

变异（Mutation）是一种基于语法的测试方法。其核心思想是：对一个已有的程序或字符串（称为基本字符串），按照明确定义的规则（称为变异运算符），进行系统性的微小修改，生成新的版本（称为变体）。

变异的通用定义： 当我们使用明确定义的规则，基于语法描述，对语法或由语法开发的对象进行系统性更改时，就是在进行变异分析。

变异的核心公式可以简单表示为：

// 变异过程
基本字符串 + 变异运算符 → 变体

// 示例
G 26 08.01.90  +  交换G/B  →  B 26 08.01.90
Formula

核心概念

定义： 文法中的一个有效字符串，作为变异的起始点。

术语 "ground" 来源于代数中的 ground term（基础项），指的是不含变量的项。类似地，基本字符串是已经被完全实例化的有效字符串。

示例：

  • G 26 08.01.90 — 一个有效的 actG 字符串
  • B 22 06.27.94 — 一个有效的 actB 字符串
  • 一个完整的 Java 程序 — 作为程序级变异的基本字符串

基本字符串用于生成有效测试；无效测试不需要基本字符串，可以直接从文法推导。

定义： 一条规则，指定由文法生成的字符串的语法变化方式。

变异运算符通常做以下两类事情之一：

  • 模拟程序员的典型错误 — 例如使用了错误的变量名、错误的运算符
  • 鼓励常见的测试启发式 — 例如使表达式为 0、使条件为 false

示例：

  • 交换动作： 将 G 替换为 B（或反之）
  • 替换数字： 将某位数字替换为其他数字
  • 算术运算符替换： 将 + 替换为 -

定义： 对基本字符串应用一次变异运算符后得到的结果。

变体是一个字符串，它要么在文法中（有效变体），要么非常接近于在文法中（无效变体）。

关键原则： 通常每次只应用一个变异运算符。多个变异可能相互干扰，降低测试效果。经验表明，单次变异更有效。

定义： 给定推导 D 的变体 m∈M 和测试 t，当且仅当 t 在 D 上的输出 ≠ t 在 m 上的输出时，称 t 杀死 m。

简单来说：如果一个测试用例能让原程序和变体产生不同的输出，那么这个测试就"杀死"了该变体。

// 杀死变体的条件
output(t, 原程序) ≠ output(t, 变体)  →  t 杀死了变体 m
Definition

变体的类型

在实际的变异测试中，产生的变体可以分为以下几种类型：

类型英文说明能否被杀死
死亡变体 Dead Mutant 已被某个测试用例杀死的变体 ✅ 已被杀死
存活变体 Live Mutant 尚未被任何测试用例杀死的变体 ⚠️ 需要更多测试
等价变体 Equivalent Mutant 与原程序行为完全相同的变体 ❌ 不可能被杀死
夭折变体 Stillborn Mutant 语法不合法的变体（无法编译） ❌ 无需杀死
平凡变体 Trivial Mutant 几乎任何测试都能杀死的变体 ✅ 容易杀死

变异测试覆盖标准

基于变异测试，可以定义三种覆盖标准：

定义： 对于每个变体 m∈M，测试需求集 TR 恰好包含一个需求 — 杀死 m。

MC 是最强的变异测试标准，要求测试集能够杀死所有非等价变体。

覆盖程度用变异分数 (Mutation Score) 衡量：

变异分数 = 被杀死的变体数 / 非等价变体总数 × 100%

定义： 对于每个变异运算符，TR 恰好包含一个需求 — 创建一个使用该运算符推导的变异字符串。

MOC 要求每个变异运算符至少被使用一次。它不要求杀死变体，只要求创建变体。

用于生成无效字符串的覆盖标准。

定义： 对于每个变异运算符，TR 包含多个需求 — 创建一个包含该运算符可变异的每个产生式的变异字符串。

MPC 比 MOC 更严格，要求每个运算符在每个可应用的产生式上都被使用。

变异测试的关键问题

❓ 是否应同时应用多个运算符？

  • 通常不需要
  • 多重变异可能相互干扰
  • 程序级变异的经验表明单次变异更有效
  • 但近年研究发现了例外情况

❓ 是否应考虑每个可能的应用？

  • 在程序级变异中是必要的
  • 每个可能的变异点都需要考虑
  • 这导致变体数量可能非常庞大
  • 需要工具支持自动化

变异测试的应用领域

变异运算符已为多种语言和领域定义：

领域语言/技术应用场景
编程语言Fortran, Lisp, Ada, C, C++, Java单元测试、集成测试
规约语言SMV, Z, Object-Z, 代数规约规约验证
建模语言Statecharts, 活动图模型测试
输入文法XML, SQL, HTML输入验证、安全测试
变异测试的本质： 变异测试不是直接生成测试用例，而是通过变体来引导测试者找到有效的测试集。如果一个测试集能杀死所有（非等价的）变体，那么它被认为是非常强大的测试集。
