变异运算符

Mutation Operators — 定义如何对程序进行系统性修改的规则集合。不同编程语言有各自的变异运算符，但方法级别的运算符在不同语言间是相似的。

AOR ROR COR SVR BSR

🐛 模拟程序员错误

变异运算符模拟程序员在编码时常见的典型错误：

  • 使用了错误的变量名
  • 使用了错误的运算符
  • 遗漏了某个语句
  • 使用了错误的常量值

🧪 鼓励测试启发式

变异运算符鼓励使用常见的测试策略：

  • 使算术表达式为 0
  • 使条件表达式为 false
  • 使数组访问越界
  • 触发异常情况

研究人员通常先设计大量的运算符，然后通过实验选择最有效的一组。

变异运算符的分类

变异运算符可以按照被修改的程序元素类型分为四大类：

📝

语句变异

Statement Mutations — 对整个语句进行修改，如删除语句、插入语句等。

🔢

运算符变异

Operator Mutations — 替换程序中的运算符，如算术、关系、逻辑运算符等。

📦

变量变异

Variable Mutations — 替换程序中的变量引用，如用其他变量替换当前变量。

💎

常量变异

Constant Mutations — 修改程序中的常量值或插入特殊函数调用。

运算符详细示例

AOR — 算术运算符替换

将一个算术运算符替换为另一个。这是最常见的变异运算符之一。

原始 
a = m * (o + p);
变体 
Δ1 a = m + (o + p);   // * → +
Δ2 a = m * (o * p);   // + → *
Δ3 a = m % (o + p);   // * → %

ROR — 关系运算符替换

将一个关系运算符替换为另一个，用于测试边界条件和比较逻辑。

原始 
if (X <= Y)
变体 
Δ1 if (X > Y)    // <= → >
Δ2 if (X < Y)    // <= → <
Δ3 if (X == Y)   // <= → ==
Δ4 if (false)     // <= → false

COR — 条件运算符替换

替换逻辑连接词（&& 和 ||），测试逻辑组合的正确性。

原始 
if (X <= Y && a > 0)
变体 
Δ1 if (X <= Y || a > 0)  // && → ||
Δ2 if (X <= Y leftOp a > 0) // 返回左操作数

SOR — 移位运算符替换

替换移位运算符，测试位操作的正确性。

原始 
b = b >> 2;
变体 
Δ1 b = b << 2;   // >> → <<
Δ2 b = b >>> 2;  // >> → >>>

LOR — 逻辑运算符替换

替换位逻辑运算符。

原始 
int c = a & b;
变体 
Δ1 int c = a | b;   // & → |
Δ2 int c = a ^ b;   // & → ^

ASR — 赋值运算符替换

替换赋值运算符，测试复合赋值的正确性。

原始 
a = m * (o + p);
变体 
Δ1 a += m * (o + p);  // = → +=
Δ2 a *= m * (o + p);  // = → *=
Δ3 a -= m * (o + p);  // = → -=

UOI — 一元运算符插入

在一元运算符位置插入取反、逻辑非等运算符。

原始 
a = m * (o + p);
变体 
Δ1 a = m * -(o + p);  // 插入 -
Δ2 a = -(m * (o + p)); // 插入 -

UOD — 一元运算符删除

删除已有的一元运算符。

原始 
if (!(X <= Y && !Z))
变体 
Δ1 if (X > Y && !Z)   // 删除外层 !
Δ2 if (!(X <= Y && Z)) // 删除内层 !

SVR — 标量变量替换

将变量替换为同一作用域中的其他变量，模拟"用错变量"的错误。

原始 
a = m * (o + p);
变体 
Δ1 a = o * (o + p);   // m → o
Δ2 a = m * (m + p);   // o → m
Δ3 a = m * (o + o);   // p → o
Δ4 p = m * (o + p);   // a → p

BSR — 炸弹语句替换

将语句替换为 Bomb()，在到达时抛出异常。用于测试代码可达性。

原始 
a = m * (o + p);
变体 
Δ1 Bomb();  // 到达时抛出异常

有效变异运算符集

并非所有变异运算符都同等重要。研究表明，一小部分运算符就能产生大部分测试效果。

有效变异运算符集的定义：
如果专门为杀死由运算符集 O = {o1, o2, ...} 创建的变体而设计的测试，也能以极高的概率杀死由所有剩余变异运算符创建的变体，则 O 定义了一个有效的变异运算符集。

这意味着：

  • 我们不需要使用所有可能的变异运算符
  • 选择最有效的子集即可获得大部分测试效果
  • 这大大降低了变异测试的成本
  • 具体的最优子集取决于编程语言和应用场景
实践建议： 在实际项目中，建议先使用一组经过验证的有效变异运算符，而不是尝试所有可能的运算符。这样可以在测试效果和成本之间取得良好的平衡。
