章节概述

软件测试是软件工程中最重要的质量保障手段。本章从"为什么要测试"出发，介绍软件故障、错误和失败的概念，以及RIPR故障模型。然后介绍测试活动的分类（测试设计、测试自动化、测试执行、测试评估）和模型驱动测试设计(MDTD)方法。最后引入覆盖标准的概念，为后续章节的图覆盖、逻辑覆盖等具体覆盖标准奠定基础。本章还包含ISTQB基础级的测试原则、测试技能和良好实践。

学习路线

本章内容按照以下路线展开，建议按顺序学习：

  1. 为什么要测试 — 理解软件故障的危害；掌握故障(Fault)、错误(Error)、失败(Failure)的概念与区别；学习RIPR故障模型的四个条件。
  2. 测试活动与模型驱动设计 — 掌握四大测试活动（设计、自动化、执行、评估）；理解模型驱动测试设计(MDTD)的流程和四种建模结构；了解测试自动化框架。
  3. 覆盖标准导论 — 理解测试需求和覆盖标准的定义；掌握覆盖标准之间的包含关系；了解生成器与识别器的概念。
  4. ISTQB测试基础 — 学习ISTQB对测试的定义和七项测试原则；掌握测试活动与测试件的关系；了解测试角色和技能要求。
  5. 实战练习 — 通过故障/错误/失败分析、RIPR模型应用、覆盖标准包含关系判断等练习，巩固本章核心知识点。

关键术语速查

Fault / Failure / Error
故障(Fault)是静态缺陷；错误(Error)是故障导致的不正确内部状态；失败(Failure)是可观察到的外部错误行为。故障→错误→失败的因果链。
RIPR 模型
故障触发失败需要四个条件：Reachability(可达性)、Infection(感染性)、Propagation(传播性)、Reveal(可观察性)。
MDTD (模型驱动测试设计)
通过将软件制品抽象为模型，从模型中提取测试需求，再生成测试用例的系统化测试设计方法。
Coverage Criterion (覆盖标准)
定义测试需求的规则和过程的集合。覆盖级别 = 已满足的测试需求 / 总测试需求。
V&V (验证与确认)
验证(Verification)检查开发产物是否满足前一阶段需求；确认(Validation)评估软件是否符合预期用途。
Test Case (测试用例)
由测试值、前缀值、后缀值、期望结果等组成的可执行测试规格。
Test Oracle (测试预言)
判断测试执行结果是否正确的机制或标准。
