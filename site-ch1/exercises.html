练习1：故障/错误/失败分析

题目描述

考虑以下计算数组中零值个数的方法：

public static int numZero(int[] arr) {
    int count = 0;
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] == 0) {
            count++;
        }
    }
    return count;
}Java

请回答以下问题：

故障： 循环的起始条件为 i = 1，应该是 i = 0。这导致数组的第一个元素 arr[0] 被跳过，不会被检查是否为零。

不执行故障意味着循环不被执行。

测试用例：arr = {}（空数组）

此时 arr.length = 0，条件 i < arr.length 即 1 < 0 为 false，循环体不执行。故障代码（i = 1）虽然存在，但不会产生影响。

执行了故障但内部状态没有错误。

测试用例：arr = {1, 2, 3}

循环执行时从 i=1 开始，跳过了 arr[0]=1。但因为 arr[0] ≠ 0，即使检查了它，count 也不会增加。所以跳过 arr[0] 不会导致 count 的值不正确——内部状态仍然是正确的。

产生错误（内部状态不正确）但输出正确。

这个情况比较微妙。实际上，一旦内部 count 值错误，返回值就会错误（因为 count 直接被返回）。所以对于 numZero 方法，error 一定会导致 failure。

但如果代码有更多后续处理（如结果经过验证或修正），理论上可能存在 error 但不 failure 的情况。对于本题，由于 count 直接返回，不存在这样的测试用例。

练习2：RIPR模型应用

题目描述

给定以下代码段，分析当输入 x = -5 时的RIPR四条件：

public static double squareRoot(double x) {
    if (x < 0) {
        return -Math.sqrt(-x);  // 故障：应抛出异常或返回NaN
    }
    return Math.sqrt(x);
}Java

Reachability（可达性）： ✅ 满足。输入 x = -5 使得条件 x < 0 为 true，执行包含故障的代码行 return -Math.sqrt(-x)。

Infection（感染性）： ✅ 满足。程序的内部状态变为不正确——返回了一个负数 -2.236... 作为 -5 的"平方根"，而数学上负数没有实数平方根。

Propagation（传播性）： ✅ 满足。不正确的内部状态传播到了输出——调用者会收到一个负数 -2.236...，这个值被传播给调用方。

Reveal（可观察性）： ✅ 满足。测试人员可以通过检查返回值 -2.236... 发现错误行为（负数不应作为平方根返回）。

结论： 四个RIPR条件全部满足，输入 x = -5 可以触发该故障导致的失败。

练习3：覆盖标准包含关系

题目描述

对于以下覆盖标准，判断它们之间的包含关系（哪些包含哪些）：

编号 覆盖标准 描述
C1 节点覆盖 (NC) 覆盖图中每个可达节点
C2 边覆盖 (EC) 覆盖图中每条可达边
C3 主路径覆盖 (PPC) 覆盖图中每条主路径
C4 口味覆盖 覆盖每种口味至少一次
C5 颜色覆盖 覆盖每种颜色至少一次
C6 口味×颜色覆盖 覆盖每种口味-颜色组合

包含关系：

  • C3 ⊇ C2 ⊇ C1：主路径覆盖 ⊇ 边覆盖 ⊇ 节点覆盖。覆盖所有主路径必然覆盖所有边，覆盖所有边必然覆盖所有节点。
  • C6 ⊇ C4：口味×颜色覆盖 ⊇ 口味覆盖。覆盖所有组合必然覆盖所有口味。
  • C6 ⊇ C5：口味×颜色覆盖 ⊇ 颜色覆盖。覆盖所有组合必然覆盖所有颜色。
  • C4 与 C5 互不包含：口味覆盖不包含颜色覆盖，颜色覆盖也不包含口味覆盖。它们是独立的覆盖维度。
  • C3 与 C4/C5/C6 无包含关系：图覆盖标准与输入域覆盖标准属于不同的建模结构，不能直接比较。

练习4：ISTQB测试原则辨析

题目1

以下哪项最准确地描述了"测试显示缺陷存在，而非不存在"这一原则的含义？

A. 测试只能找到所有缺陷的一部分
B. 通过测试可以证明软件没有缺陷
C. 测试可以证明存在缺陷，但不能证明没有缺陷
D. 测试人员应该尽量多地发现缺陷

答案：C — 测试可以证明缺陷存在（发现了缺陷），但不能证明缺陷不存在（测试未发现缺陷不代表没有缺陷）。

题目2

"杀虫剂悖论"对应的是ISTQB七项测试原则中的哪一项？

A. 穷尽测试不可能
B. 缺陷聚集在一起
C. 测试会磨损
D. 测试取决于上下文

答案：C — "测试会磨损"（Testing wears out）即杀虫剂悖论：反复使用相同的测试用例，最终将不再发现新的缺陷。需要定期修改和更新测试用例。

题目3

在一个安全关键的医疗设备软件项目中，以下哪种测试独立性级别最为合适？

A. 开发人员测试自己的代码
B. 同一团队的其他开发人员测试
C. 组织内独立的测试团队测试
D. 外部认证机构进行独立测试

答案：D — 对于安全关键系统（如医疗设备），通常需要最高级别的测试独立性。外部认证机构的测试可以提供最客观的评估，符合监管要求。实践中可能选择C并辅以D。

题目4

Verification（验证）和Validation（确认）的区别是什么？以下哪个描述最准确？

A. 验证是动态测试，确认是静态测试
B. 验证检查是否正确构建产品，确认检查是否构建了正确的产品
C. 验证在开发阶段进行，确认在测试阶段进行
D. 验证由开发者执行，确认由用户执行

答案：B — Verification = "Are we building the product right?"（是否正确地构建产品），Validation = "Are we building the right product?"（是否构建了正确的产品）。哈勃望远镜案例完美说明了二者的区别。
