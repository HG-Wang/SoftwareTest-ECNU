覆盖标准导论

Coverage Criteria — 测试需求、覆盖标准、包含关系以及生成器与识别器的概念。

覆盖标准 测试需求 包含关系

输入空间与测试选择

即使是小程序也有巨大的输入空间。考虑一个在32位机器上接受3个int参数的程序：

输入空间大小： 232 × 232 × 232 = 296 ≈ 7.9 × 1028 种输入组合。如果每秒测试10亿种组合，需要约 25亿年 才能穷举测试完毕。穷尽测试对于任何非平凡的软件来说都是不可能的。

因此，我们需要一种结构化的方式来搜索输入空间，选择最有价值的测试用例。这就是覆盖标准（Coverage Criteria）的作用——它们帮助我们在有限的测试预算内最大化缺陷发现能力。

测试需求与覆盖标准

Test Criterion（测试标准）
定义测试需求的规则和过程的集合。每个覆盖标准都规定了测试应该覆盖哪些元素（节点、边、路径、变量定义等）。
Test Requirements（测试需求）
测试中必须满足或覆盖的具体事物。例如，"覆盖每个节点"是一个测试需求，"覆盖每个定义到使用的路径"也是一个测试需求。
Coverage Level（覆盖级别）
已满足的测试需求占总测试需求的比率。Coverage Level = |已满足的TR| / |总TR|。覆盖级别为1.0（100%）表示所有测试需求都已满足。

四种建模结构对应的覆盖标准

建模结构 测试需求类型 覆盖标准示例
Input Domains（输入域） 输入值的分区或等价类 等价类覆盖、边界值覆盖
Graphs（图） 图中的节点、边、路径 节点覆盖、边覆盖、主路径覆盖
Logical Expressions（逻辑表达式） 谓词中的子表达式和条件 谓词覆盖、子句覆盖、CACC
Syntax Descriptions（语法描述） 语法规则的产生式 产生式覆盖、变异覆盖

包含关系

覆盖标准之间可以存在包含关系（Subsumption），这是衡量覆盖标准强弱的重要指标：

定义： 覆盖标准 C₁ 包含（subsume） C₂，当且仅当满足 C₁ 的每一个测试集也满足 C₂。记作 C₁ ⊇ C₂。

包含关系的含义

如果 C₁ ⊇ C₂，则：

  • C₁ 是比 C₂ 更强的标准
  • 满足 C₁ 自动保证满足 C₂
  • C₁ 的测试需求是 C₂ 的测试需求的超集
  • C₁ 通常需要更多的测试用例

示例

🍬 果冻豆示例

6种口味 × 4种颜色 = 24种果冻豆。

  • 口味覆盖： 每种口味至少尝1次（6个测试需求）
  • 颜色覆盖： 每种颜色至少尝1次（4个测试需求）
  • 口味×颜色覆盖： 每种组合至少尝1次（24个测试需求）

口味×颜色覆盖 ⊇ 口味覆盖，口味×颜色覆盖 ⊇ 颜色覆盖。但口味覆盖不包含颜色覆盖，反之亦然。

📐 图覆盖示例

对于图结构的覆盖标准：

  • 边覆盖 ⊇ 节点覆盖
  • 主路径覆盖 ⊇ 边覆盖
  • 主路径覆盖 ⊇ 节点覆盖

覆盖了所有边必然覆盖了所有涉及的节点，因此边覆盖包含节点覆盖。

生成器与识别器

对于覆盖标准，存在两个关键问题：

🔧 Generator（生成器）

自动生成满足准则的值的程序。

  • 输入：一个覆盖标准和被测软件
  • 输出：一组满足该标准的测试用例
  • 对于大多数准则，完全自动化的生成是不可判定的

🔍 Recognizer（识别器）

判断给定测试集是否满足准则的程序。

  • 输入：一个测试集和一个覆盖标准
  • 输出：是/否（是否满足该标准）
  • 对于大多数准则，识别比生成容易得多
关键洞察： 对于大多数覆盖标准，生成和识别两个问题在理论上都是不可判定的（undecidable）——即不存在通用算法能够对所有程序给出确定的答案。但在实践中，识别比生成容易得多。这就是为什么覆盖分析工具（识别器）相当丰富，而自动化测试生成工具（生成器）仍在不断发展中。

小示例：indexOf 方法的图覆盖

public int indexOf(Node n) {
    int index = 0;
    Node current = head;
    while (current != null) {
        if (current.equals(n)) {
            return index;
        }
        index++;
        current = current.next;
    }
    return -1;
}Java

对于这个方法：

  • 节点覆盖要求每个语句至少被执行一次
  • 边覆盖要求每个控制流分支至少被执行一次
  • 识别器可以检查测试路径是否遍历了所有节点/边
  • 生成器需要自动构造遍历所有节点/边的输入
