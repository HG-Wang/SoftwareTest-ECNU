语义逻辑覆盖标准

Semantic Logic Coverage Criteria — 基于谓词和子句的语义定义，从最基础的谓词覆盖到强大的 ACC 家族，系统地定义逻辑表达式的测试标准。

8.1 PC CC CoC ACC MC/DC

基本概念

谓词 (Predicate)

谓词是一个返回布尔值的逻辑表达式，由多个子句通过逻辑运算符（∧、∨、¬ 等）连接而成。在程序中，谓词通常出现在 if、while、for 等控制流语句中。

形式化定义： 谓词 p 是一个函数 p: {T, F}^n → {T, F}，其中 n 是子句的数量。

子句 (Clause)

子句是谓词中的基本布尔变量或取反的布尔变量。子句的取值只能为 true 或 false。

示例

考虑谓词 p = a ∧ (b ∨ c)：

组成部分类型说明
p谓词整个逻辑表达式
a子句第一个合取项
b子句第二个合取项的第一个析取项
c子句第二个合取项的第二个析取项

实证统计

工业实践中的统计： 约 88.5% 的谓词仅包含 1 个子句（即简单布尔变量），约 9.4% 包含 2 个子句，仅 2.1% 包含 3 个或更多子句。这意味着大多数情况下，PC 和 CC 就能满足需求。

基础覆盖标准

三种基础覆盖标准从简单到严格递进：

Predicate Coverage (PC)

定义： 对于谓词 p，测试集必须包含使 p = true 和 p = false 的测试用例各至少一次。

PC 是最基本的逻辑覆盖标准，等价于分支覆盖（Branch Coverage）。它只要求谓词整体取到两个值。

真值表示例：p = a ∧ b

测试abp = a ∧ bPC 满足？
t₁TTT✓ p=T
t₂FTF✓ p=F

测试集 {t₁, t₂} 即可满足 PC。

PC 的局限性： PC 不保证每个子句都取到两个值。例如上面的测试中，b 始终为 T，如果 b 的实现有误（恒为 true），PC 无法发现。

Clause Coverage (CC)

定义： 对于谓词 p 中的每个子句 cᵢ，测试集必须包含使 cᵢ = true 和 cᵢ = false 的测试用例各至少一次。

CC 要求每个子句都取到两个值，但不要求在同一测试中同时观察谓词的变化。

真值表示例：p = a ∧ b

测试abp = a ∧ bCC 满足？
t₁TTTa=T✓ b=T✓
t₂FFFa=F✓ b=F✓

测试集 {t₁, t₂} 满足 CC，每个子句都取到了 T 和 F。

CC 的局限性： CC 不保证谓词随子句变化而变化。例如 p = a ∨ ¬a，无论 a 取何值 p 恒为 true，但 CC 仍可被满足。这意味着 CC 不能保证子句对谓词有"影响力"。

Combinatorial Coverage (CoC)

定义： 对于谓词 p 中的 N 个子句，测试集必须覆盖所有 2^N 种子句取值组合。

CoC 是最严格的语义覆盖标准，等价于真值表的每一行都被测试到。它自然包含 PC 和 CC。

真值表示例：p = a ∧ b

测试abp = a ∧ bCoC 行
t₁TTT行 1
t₂TFF行 2
t₃FTF行 3
t₄FFF行 4
CoC 的局限性： 测试数量随子句数指数增长（2^N）。当 N=10 时需要 1024 个测试，实际中往往不可行。ACC 家族正是为了解决这个问题而提出的。

有效子句覆盖 ACC

核心思想

ACC（Active Clause Coverage）的核心思想是：一个子句应该是"有效的"（active），即它能够决定整个谓词的取值。换句话说，当主子句改变取值时，谓词也应该随之改变。

决定 (Determination) 的定义： 子句 cᵢ 在赋值 a 下决定谓词 p，当且仅当：
p[a] ≠ p[a[cᵢ→¬a(cᵢ)]]
即：保持其他子句不变，仅翻转 cᵢ 的取值，谓词的值发生了变化。

决定示例

对于 p = a ∨ b：

abp = a ∨ b翻转 a新 pa 决定 p？
TTTFT否（p 不变）
TFTFF是（p 改变）
FTTTT否（p 不变）
FFFTT是（p 改变）

可以看到，a 仅在 b = F 时能决定 p。

ACC 的三个变体

ACC 有三个变体，区别在于对次子句取值的约束以及是否要求谓词改变：

定义： 对于谓词 p 中的每个子句 cᵢ，存在两个测试 a, b 使得：
1. cᵢ 在 a, b 中取值不同
2. cᵢ 在赋值 a 下决定 p
3. 次子句的取值可以不同（无约束）

GACC 是最弱的 ACC 变体。它不要求次子句在两个测试中保持一致，也不要求谓词必须改变。

问题： GACC 不保证满足 PC。因为次子句取值不同，谓词可能在两次测试中恰好同为 true 或同为 false。

定义： 对于谓词 p 中的每个子句 cᵢ，存在两个测试 a, b 使得：
1. cᵢ 在 a, b 中取值不同
2. cᵢ 在赋值 a 下决定 p
3. 次子句的取值可以不同（无约束）
4. p(a) ≠ p(b) — 谓词必须改变

CACC 在 GACC 的基础上增加了谓词必须改变的要求。次子句可以取不同值，只要谓词最终改变即可。

CACC 保证满足 PC，因为谓词在两个测试中取值不同。

定义： 对于谓词 p 中的每个子句 cᵢ，存在两个测试 a, b 使得：
1. cᵢ 在 a, b 中取值不同
2. cᵢ 在赋值 a 下决定 p
3. 次子句的取值必须相同（a(d) = b(d)，对所有 d ≠ cᵢ）
4. p(a) ≠ p(b) — 谓词必须改变

RACC 是最强的 ACC 变体。它要求次子句取值完全一致，只有主子句不同，且谓词随之改变。

RACC 等价于 MC/DC（修改条件/判定覆盖），是航空安全标准 DO-178B Level A 软件的要求。

✓ RACC 包含 PC

RACC 要求谓词必须改变，因此自然满足 PC（谓词取到 T 和 F）。

✗ GACC 不包含 PC

GACC 不要求谓词改变，可能两个测试中谓词都为 T 或都为 F。

✗ CC 不包含 PC

CC 保证每个子句取到 T 和 F，但不保证谓词也取到两个值。

✓ CoC 包含一切

CoC 覆盖所有 2^N 种组合，自然满足所有其他标准。

实践指导

根据谓词中子句的数量，推荐使用不同的覆盖标准：

子句数 N推荐标准测试数上界理由
N = 1 PC 2 只有一个子句，PC 即可覆盖所有情况。
N = 2 RACC 4 测试数量可控，RACC 提供最强保障。
N = 3~5 CACC 2N RACC 可能不总是可行，CACC 是很好的折中。
N = 6~10 GACC 2N 子句较多时，使用较弱但实用的 GACC。
N > 10 PC 2 子句极多时，建议先重构简化谓词，再用 PC。
最佳实践： 如果谓词包含太多子句（N > 5），通常说明代码需要重构。将复杂条件拆分为多个简单条件，不仅可以降低测试难度，还能提高代码可读性和可维护性。
注意： 以上测试数上界是理论最大值。实际中，很多覆盖标准的测试需求可以被同一个测试用例同时满足，因此实际需要的测试数通常远小于上界。
