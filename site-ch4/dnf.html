DNF 语法覆盖标准 (Syntactic Logic Coverage)

基于析取范式 (Disjunctive Normal Form) 的语法结构，利用蕴含项和文字的组合定义覆盖标准，具有更强的故障检测能力。

§ 8.2 DNF MUMCUT

DNF 基本概念#

字 (Literal)
一个布尔变量或其否定。例如：a、¬b 都是字。
项 / 蕴含项 (Implicant / Term)
若干字的合取（AND）。例如：ab 表示 a ∧ b，a¬bc 表示 a ∧ ¬b ∧ c。
析取范式 (DNF)
若干蕴含项的析取（OR）。例如：abc + ab + ac 表示 (a∧b∧c) ∨ (a∧b) ∨ (a∧c)。
DNF 的标准写法中，省略 ∧ 符号，用并列写字母表示合取，用 + 表示析取。例如 ab + cd 表示 (a∧b) ∨ (c∧d)。

蕴含项覆盖 IC (Implicant Coverage)#

IC 要求对 DNF 中的每个蕴含项（term），选择一个使其为 True 的取值，且该蕴含项中的否定字也需被覆盖。

例如对于 f = ab + cd：

  • 蕴含项 ab：选择 a=T, b=T
  • 蕴含项 cd：选择 c=T, d=T

关键定义#

蕴含项 ti 的真子项是通过去掉 ti 中一个或多个字后得到的项。例如 a 是 ab 的真子项。

一个蕴含项 p 是素蕴含项，当且仅当不存在其他蕴含项 q 使得 q 的真子项包含 p 且 q 也蕴含 f。简而言之，素蕴含项不能再被扩大而仍然蕴含函数。

如果去掉 DNF 表达式中的某个蕴含项后，函数值不变，则该蕴含项是冗余的。例如 f = a + ab 中，ab 是冗余的，因为 a 已经覆盖了所有使 ab 为真的情况。

不含冗余蕴含项的 DNF 表达式。所有蕴含项都是素蕴含项，且无冗余。这是最简洁的等价 DNF 表示。

UTP 与 MUTP#

UTP (Unique True Point)

给定蕴含项 ti，一个 UTP 是一组赋值，使得：

  • ti 为 True
  • 所有其他蕴含项 tj（j ≠ i）均为 False

MUTP (Multiple Unique True Points)

对每个蕴含项 ti，选择 UTP，其中 不在 ti 中的字（non-in-clause literals）需在不同 UTP 中分别取 T 和 F。

例如 f = ab + cd：

蕴含项UTP非蕴含项中的字取值
ab {TTFF, TTFT, TTTF} c, d 各取 T/F
cd {FFTT, FTTT, TFTT} a, b 各取 T/F

NFP 与 CUTPNFP#

NFP (Near False Point)

给定蕴含项 ti 和其中的字 c，一个 NFP 是一组赋值，使得：

  • f 为 False
  • 将 c 取反后，f 变为 True

这意味着 c 在该点决定 (determines) f。

CUTPNFP (Coupled UTP-NFP)

对每个蕴含项 ti 中的每个字 c，选择一个 UTP 和一个 NFP 配对，使得两者仅在字 c 的取值上不同。

MNFP (Multiple Near False Points)#

对每个蕴含项 ti，选择 NFP，其中不在 ti 项中的字需在不同 NFP 中分别取 T 和 F。

MUMCUT 覆盖标准#

MUMCUT = MUTP + CUTPNFP + MNFP

MUMCUT 是 DNF 最全面的语法覆盖标准，结合了三种测试选择策略，能够检测所有 9 类 DNF 故障。

与其他标准的对比

MUMCUT

  • 测试用例数量较多（高于 RACC）
  • 检测 全部 9 类 DNF 故障
  • 故障检测率约为 RACC 的 3 倍
  • 适用于已知 DNF 结构的规约

RACC

  • 测试用例数量较少
  • 检测部分 DNF 故障
  • 不依赖 DNF 语法结构
  • 更通用的语义覆盖标准

Karnaugh Map 应用#

Karnaugh Map (卡诺图) 在 DNF 逻辑覆盖中有多种应用：

  • 子句决定关系判定：通过卡诺图中相邻格的变化判断子句是否决定谓词
  • 取反分析：在卡诺图上标注取反后的函数值变化
  • 素蕴含项识别：卡诺图中最大的矩形区域即为素蕴含项
  • UTP 定位：只被一个蕴含项覆盖的格即为 UTP
  • UTP/NFP 配对：相邻的 UTP 和非覆盖格构成 CUTPNFP 对
